Machina
Repository: github.com/ri5hii/machina

Concurrent job execution engine for Go

Machina decouples work submission from execution, allowing systems to handle slow, heavy, or batch workloads without blocking the caller.

Instead of executing work inline:
Client → Submit Job → Receive ID immediately

The engine handles:
queueing → scheduling → execution → result storage

This enables asynchronous, resource-controlled processing with bounded concurrency and structured lifecycle management.

Designed for systems where reliability, control, and scalability of background work matter.

System Design

  • Two-tier concurrency model:
    • - Worker pool (bounded job-level parallelism)
    • - Chunked fan-out using errgroup (item-level parallelism)
  • Strict separation of concerns:
    • - Jobs define logic
    • - Engine handles execution
    • - Registry maps job types
    • - Constructors bridge JSON → typed jobs
  • Context-driven cancellation ensures all in-flight work stops on failure
  • No unbounded goroutines regardless of workload size

Execution Model

  • Job lifecycle:
    • pending → running → completed / failed
  • Batch jobs:
    • - Scan → discover items
    • - Chunk → partition workload
    • - RunBatch → parallel execution
    • - Aggregate → final result merge
  • Failure propagation via shared context cancellation

Job Contracts

  • SingleRunJob: Run(ctx)
  • BatchProcessingJob:
    • - Scan()
    • - ChunkSize()
    • - RunBatch(ctx, batch)
    • - Aggregate(results)
  • Jobs never manage goroutines or scheduling

API & CLI

  • API:
    • - POST /jobs → submit
    • - GET /jobs → list
    • - GET /jobs/:id → status
    • - GET /health → health check
  • CLI:
    • - start, shutdown, health
    • - submit, status, list
    • - register, unregister, types
    • - config management

Built-in Jobs

  • file_encrypt → AES-256-GCM folder encryption
  • csv_transform → row-level transformations

Configuration

  • - port (default: 8080)
  • - workers (default: 4)
  • - queue-size (default: 100)
  • - log-level (default: INFO)
  • - precedence: flag → config → fallback
