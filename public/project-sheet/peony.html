Peony
Repository: github.com/ri5hii/peony

A calm, CLI-first cognitive holding space

Peony is a local-first system designed for unfinished thoughts — ideas that are not ready to become tasks, notes, or decisions.

Instead of forcing structure, Peony allows thoughts to exist in a natural lifecycle:
capture → rest → resurface → evolve or release

It removes urgency, metrics, and pressure, replacing them with time-aware resurfacing and gentle interaction.

Built for clarity, reflection, and psychological safety — not productivity.

Philosophy

  • - Thoughts are seeds, not obligations
  • - They may rest, mature, transform, or dissolve
  • - No deadlines, no reminders, no urgency loops
  • - Language is soft — no streaks, no metrics, no guilt
  • - Private by default: no accounts, no tracking

Cognitive Model

  • Core unit: Cognitive Unit (CU)
  • Represents:
    • - unresolved decisions
    • - early ideas
    • - questions
    • - memory fragments
  • Contains:
    • - lifecycle state
    • - temporal context
    • - optional emotional metadata
    • - interaction history

Lifecycle

  • - Captured → Resting → Tended
  • - → Evolved → Released → Archived
  • - No failure states
  • Resurfacing is time-aware, not scheduled

CLI Experience

  • - add → capture thought
  • - tend → surface reflections
  • - view → contextual revisit
  • - rest → defer intentionally
  • - evolve → externalize
  • - release → let go
  • - archive → preserve
  • Designed to feel slow, intentional, and non-mechanical

Frontend

  • Optional, read-only interface
  • Visualizes thought lifecycles
  • No dashboards, metrics, or urgency signals
  • Supports reflection without interaction pressure

Architecture

  • Core Engine (Go)
  • CLI (Bubble Tea + Lip Gloss)
  • Storage (SQLite, local-first)
  • Frontend (GoTH + HTMX + Tailwind)

Status & Roadmap

  • CLI complete, frontend pending
  • v0.6 → read-only frontend
  • v0.7 → refinement
  • v1.0 → optional reflective AI
  • AI will be opt-in, reflective, non-prescriptive
